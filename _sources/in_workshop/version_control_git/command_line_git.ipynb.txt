        "## **Best Practices For Git Repos**\n",
        "\n",
        "- Documentation: Every repo should have a README.md.\n",
        "- Committing: Commit often, once for each new task you accomplish. If a single commit includes tons of changes, then you will struggle to understand them or undo them.\n",
        "- Pushing: Push whenver you get the chance. Any code that isn't pushed can still be lost.\n",
        "- Licensing: When you are making code that you think other people might use, consider adding a license (GitHub has licenses you can choose from).\n",
        "- References: In addition to licensing, code that supports research can have related DOI or Zenodo links\n",
        "- Security: Use SSH whenever you can (not https), because it is a more secure protocol.\n",
        "- Large Files: Do not store large files on git, always have a .gitignore. If you need to manage large files look here: [Git Large File Storage](https://docs.github.com/en/repositories/working-with-files/managing-large-files/about-git-large-file-storage)"
